# Історія змін

## 2026-06-17

### Додано

- Додано налаштування розміру для блоків зображення.
- Додано налаштування розміру для блоків відео.
- Додано запит розміру при вставці зображення у WYSIWYG-блок через URL.
- Додано блок `Секція / колонки`.
- Додано можливість створювати секції з 1–4 колонками.
- Додано налаштування ширини колонок у відсотках.
- Додано можливість додавати блоки всередину колонок секції.

### Виправлено

- Маркер або номер списку у WYSIWYG тепер рухається разом із текстом при вирівнюванні.
- Усунуто дублювання тексту в Tiptap WYSIWYG-блоці після збереження та повторного відкриття редактора.

### Раніше

- Додано WYSIWYG-блок на базі Tiptap для текстових блоків конструктора сторінок.

## 2026-06-17

### Виправлено

- Виправлено збереження секцій/колонок у конструкторі сторінок.
- Серверна валідація тепер приймає блок SECTION і перевіряє вкладені блоки в колонках.

### Конструктор сторінок

- Верхній рівень контенту переведено на модель секцій.
- Єдина дія додавання на сторінці — «Додати секцію».
- Нова секція створюється з однією колонкою шириною 100%.
- Контентні блоки додаються всередину колонок секцій.

## 2026-06-17

### Додано

- Додано першу реалізацію типу завдання `OLYMPIAD`.
- Додано налаштування кількості асоціацій і рівнів для олімпійки.
- Додано автоматичний розрахунок кількості клітинок олімпійки.
- Додано редактор клітинок олімпійки.
- Додано умови завершення олімпійки: верхня клітинка, всі коди або N будь-яких кодів.
- Додано табличне відображення олімпійки для гравця.
- Додано відкриття клітинок після введення правильних кодів.
- Додано налаштування приховування блоку «Відповіді».

### Виправлено

- Назва завдання тепер може бути порожньою.
- Якщо назва завдання порожня, гравець бачить тільки `Завдання N.`.

## 2026-06-17

### OLYMPIAD v2

- Додано окремі налаштування купівлі кодів в олімпійці.
- Додано поле «Можливість купівлі кодів через N секунд».
- Додано поле «Купівля» для кожної клітинки олімпійки.
- Для гравця додано таймер появи можливості купувати коди.
- Після завершення таймера в закритих клітинках зʼявляється кнопка `Купити (+N сек)`.
- Додано підтвердження купівлі коду.
- Після купівлі код відкривається у відповідній клітинці, а команді нараховується штраф часу.
- Для нижнього рівня олімпійки додано редагування вмісту клітинки через конструктор контенту.
- Вміст нижніх клітинок у таблиці гравця масштабується до розміру клітинки, а не обрізається.
- Клік по масштабованому вмісту відкриває повний перегляд клітинки у попапі.

## 2026-06-17 — виправлення OLYMPIAD v2

- Прибрано блок «Додати коди» для завдань типу OLYMPIAD.
- Поле «N кодів для завершення» тепер показується тільки для умови «здати N будь-яких кодів».
- Виправлено поведінку попапу перегляду клітинки олімпійки.
- Купівля коду тепер повертає повідомлення «Код ... прийнято».
- Купівля верхнього коду коректно завершує олімпійку за умовою «верхня клітинка».
- Штрафи за купівлю кодів тепер потрапляють у статистику як ручні штрафні коригування з коментарем про купівлю.

## 2026-06-17

### OLYMPIAD

- Ущільнено відображення нижніх клітинок олімпійки.
- Прибрано зайвий порожній простір навколо масштабованого вмісту клітинок.
- Додано окремий стиль для кнопки «Редагувати вміст».
- Перебудовано блок налаштувань олімпійки: кожне налаштування в окремому рядку.

### Додано перевірку дублювання кодів

- У редакторі завдань додано перевірку однакових кодів перед збереженням.
- Якщо два коди збігаються, збереження блокується без перезавантаження сторінки.
- Повідомлення показує номери кодів, які потрібно виправити.
- Перевірка працює для стандартних завдань і для OLYMPIAD.


## Поточне виправлення

- Повідомлення про дублікати кодів у редакторі завдань тепер показується через alert.
- Виправлено визначення номерів дублікатів у звичайних завданнях.
- При помилці збереження кодів сторінка не перезавантажується.

## 2026-06-17

### MULTITASK

- Додано тип завдання `MULTITASK`.
- Додано окремий режим редактора з вкладками `Multitask` і `Автоперехід`.
- Додано створення підзавдань із власним контентом, кодом, описом, коментарем і підказкою.
- Додано умову проходження Multitask: всі коди або N кодів.
- Додано підказки підзавдань: без підказки, підказка за часом, платна підказка.
- Додано купівлю підказок зі штрафом у секундах.
- Після здачі правильного коду підзавдання замінюється зеленим прийнятим значенням.
- Покращено повідомлення про дублікати кодів: показується значення дубля і тип/номер коду.

## 2026-06-17 — виправлення MULTITASK і редактора

- Поле N для умови проходження MULTITASK тепер приховується, якщо не обрано режим «Потрібно здати N кодів».
- Поля налаштування підказки в MULTITASK показуються тільки для відповідного типу підказки.
- Прибрано окрему кнопку «Зберегти Multitask» — збереження виконується загальною кнопкою сторінки.
- Додано кнопку «Попередній перегляд» поруч із загальною кнопкою збереження завдання.
- Блок дій редактора сторінок і завдань закріплено внизу екрана.
- Уточнено логіку групового переведення на фініш: команда з активним незавершеним завданням не пропускається лише через некоректно встановлений `finished_at`.
